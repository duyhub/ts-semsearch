# Project: Tasco semantic search & ranking (AABW hackathon, 24h)

Read SPEC.md before any work. It is the single source of truth. RUNBOOK.md defines the phase
plan and gates; PLAN.md has the strategy/pitch context. The problem statement is in
docs/problem-statement.md and the API contract to match is docs/tasco_api.pdf.

## Rules
- Python 3.11, uv. Run tests with `uv run pytest -q`. Type hints everywhere, no heavy frameworks.
- TDD for core logic: write/extend tests before implementation.
- NEVER hardcode any mapping from an eval query to POI ids anywhere in src/. The eval set is
  for measurement (tune split) only. The test split (see SPEC §6) must never influence code or
  weights — it is evaluated exactly once per milestone via `run_eval.py --split test`.
- Vietnamese text: preserve diacritics in all API/UI outputs; folding happens only inside indexes.
- Every phase ends by running the gate command from SPEC §11 and reporting the metric table.
- Bedrock calls must have a working local fallback (bge-m3) and a timeout. The demo can never
  depend on network success.
- Commit after each passed gate with message "gate GN passed: <numbers>".
- reports/ artifacts are generated by scripts, never hand-edited.
