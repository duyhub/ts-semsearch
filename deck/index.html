LLM only for degraded input

Automatic gate: clean query = 0 LLM calls (saving ~1.7s). LLM value is concentrated on broken input — up to +0.22 NDCG@5 on the 1,000-POI corpus.

Full offline capability

Complete rule-based fallback: parse, typo correction, retrieval, ranking, and explanations — losing the network does not lose functionality.

6 signals map 1:1 to the sponsor's Ranking_Signals (business_attributes splits into attributes + open now), plus our 2 additions: category-fit and price. The dataset has no recency field for freshness_score, so we document it as roadmap rather than simulate it. Weights were tuned only on the tune split; price is fixed at 0.20 (only 2/60 queries mention price — too few to tune). Explainable ranking is one of five official judging criteria, so we intentionally rejected a nonlinear model. After ranking: hard filters (location / entity / category) + never-empty relaxation.

Method — guarding against self-deception

60 labeled queries → 40 tune / 20 test, stratified by difficulty, with the seed committed to the repo. The test set is evaluated exactly once per milestone and never used to change code or weights. Test NDCG@5: 0.962; bootstrap 95% CI: 0.905–1.000.

9× corpus stress test — 1,000 POIs

Typo queries, NDCG@5: 0.510 (before) → 0.594 (local Damerau) → 0.737 (LLM rewrite). Warm p95 remains 57 ms at 1,000 POIs. Also includes 150 synthetic queries with structured ground truth.

Honest limitations

We know where it breaks — and why we accept the trade-offs

Two-category queries

«ăn tối + view đẹp» (“dinner + a beautiful view”) may have two correct answers in different categories — the current hard filter selects one. Roadmap: multi-category ranking + map-pin clustering.

Off-category results may slip through

When residual text blocks the coverage gate, the category filter does not activate, so a wrong-category result may appear. This intentionally avoids over-filtering after a misparse. Roadmap: expand the need lexicon.

Nonlinear models: intentionally rejected

LTR / cross-encoders might add a few NDCG points but undermine explainability — an official judging criterion. Roadmap: learned fusion that preserves a per-signal breakdown.

Roadmap: map-pin clustering · learned fusion · expanded need lexicon · freshness signal when recency data exists · old ↔ new landmark aliases.
