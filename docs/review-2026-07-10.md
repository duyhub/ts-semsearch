## Accepted as debt (hackathon window)

One line each — known, low-risk for the demo, deferred deliberately.

| Item | Why deferred |
|---|---|
| C26 UI fallback-notice unreachable | The honest "popular places nearby" notice branch never renders because the UI reaches results before the fallback state; API-side labeling (`meta.source`) is correct, so the demo stays honest. UI-only polish. |
| UI `districtOf` label parsing | Best-effort string parse of the district label; cosmetic, not correctness. |
| JS 25 km vs Python 30 km map radius | The UI map's nearby radius (25 km) diverges from the server anchor gate's tight radius (30 km); purely a map-view nicety. |
| `run()` race guard (UI) | Rapid re-queries can interleave; last-write-wins is acceptable for a single-operator demo. |
| `X-Request-Id` / UI polish | Minor UI header/telemetry polish. |
| `split.py` per-stratum rounding | Stratified split uses per-stratum `round()`; edge totals could shift by ±1 under a different seed — the committed seed is fixed and verified 40/20. |
| Dense `argsort` tie instability | `np.argsort` is not stable on exact-tie cosines; ties are vanishingly rare on 1024-d vectors and do not affect gates. |
| `DenseIndex` ValueError → no rebuild | A manifest mismatch raises instead of auto-rebuilding; correct-by-refusal (A2), rebuild is a manual `ingest.py` step. |
| `qcache` unbounded growth | The on-disk query-embed cache is never evicted; negligible at demo scale. |
